## How I work

I like building small but complete systems: clear user flow, real API integration, safe handling of credentials, and documentation that another person can run without guessing.

For portfolio and work samples, I separate public implementation patterns from private client data. Public repos use sample inputs, dummy IDs, and generic business scenarios.

## Current portfolio direction

- Strengthen public repositories as implementation proof, not just code storage
- Extract reusable templates from past automation work
- Keep private client files, contracts, credentials, and personal career materials out of public GitHub
